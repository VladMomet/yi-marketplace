# 🚀 Запуск Yí на Vercel — пошаговая инструкция

> Эта инструкция написана с расчётом, что вы **не программист**. Просто следуйте шагам по порядку, не пропуская. Время на всё: **40-60 минут**.

## ❓ Что мы будем делать (общая картина)

Мы создадим аккаунты в трёх бесплатных сервисах:
1. **GitHub** — хранилище кода (как Google Drive, но для программистов)
2. **Vercel** — хостинг сайта (берёт код из GitHub и разворачивает в интернете)
3. **Neon** — облачная база данных PostgreSQL

Плюс настроим:
- **Telegram-бот** — для уведомлений менеджеру

После этого ваш сайт будет доступен по адресу `https://yi-marketplace-XXXX.vercel.app` (потом можно подключить свой домен).

**Всё бесплатно** на старте.

⚠️ **Важно про 152-ФЗ:** на Vercel сервера в Европе. Это нормально для тестирования и демо, но **нельзя принимать реальных клиентов с реальными персональными данными** в этой конфигурации. Когда соберётесь принимать первых клиентов — переезжаете на VPS в РФ. До тех пор можно показывать сайт партнёрам и тестировать UX.

## ШАГ 1. Создать GitHub-аккаунт и репозиторий

GitHub — это место, где будет лежать код вашего проекта.

### 1.1. Регистрация

1. Откройте [github.com](https://github.com/)
2. Нажмите **Sign up** (правый верхний угол)
3. Введите email, придумайте пароль, выберите username (например `vladimir-obolenskiy`)
4. Подтвердите email — на почту придёт письмо с кодом

### 1.2. Создать новый репозиторий

1. После входа нажмите зелёную кнопку **New** (или плюсик в правом верхнем углу → **New repository**)
2. Заполните:
   - **Repository name:** `yi-marketplace`
   - **Description:** `B2B-маркетплейс мебели из Китая` (необязательно)
   - **Private** — выберите **Private** (приватный, чтобы никто посторонний не видел код)
   - Остальные галочки не трогайте
3. Нажмите **Create repository**

Откроется страница с инструкциями типа "...or push an existing repository". Запомните этот URL — он понадобится. Выглядит примерно так:
```
https://github.com/vladimir-obolenskiy/yi-marketplace.git
```

### 1.3. Залить код проекта

Самый простой способ — через **GitHub Desktop** (приложение, не нужен терминал).

1. Скачайте [GitHub Desktop](https://desktop.github.com/) и установите
2. Откройте, войдите тем же аккаунтом
3. **File → Clone repository → URL** → вставьте URL вашего репозитория → выберите папку на компьютере куда сохранить (например `~/Documents/yi-marketplace`)
4. Нажмите **Clone**
5. Распакуйте `yi-marketplace.zip` (который я вам дал) во **временную** папку
6. **Скопируйте всё содержимое распакованной папки** (но не саму папку — а файлы внутри неё: `src/`, `package.json`, `.env.example` и т.д.) в папку, которую вы клонировали через GitHub Desktop
7. Вернитесь в GitHub Desktop — он автоматически увидит новые файлы
8. В левом нижнем углу впишите Summary: `Initial commit` → нажмите **Commit to main**
9. Сверху нажмите **Push origin**

Обновите страницу репозитория на github.com — там должны появиться файлы.

✅ **Готово!** Код теперь на GitHub.

## ШАГ 2. Создать базу данных в Neon

Neon — это бесплатный PostgreSQL в облаке. Бесплатного плана достаточно для старта.

### 2.1. Регистрация

1. Откройте [neon.tech](https://neon.tech/)
2. **Sign up** → войдите через GitHub (так проще всего)

### 2.2. Создать проект

1. После входа Neon предложит создать первый проект:
   - **Project name:** `yi-marketplace`
   - **Postgres version:** последняя (по умолчанию)
   - **Region:** `Europe (Frankfurt)` — самый близкий к РФ
2. Нажмите **Create project**

### 2.3. Скопировать строку подключения

После создания проекта Neon покажет **Connection string** — длинную строку вида:
```
postgresql://yi_owner:abcdef123@ep-cool-name-12345.eu-central-1.aws.neon.tech/neondb?sslmode=require
```

**🚨 Скопируйте эту строку и сохраните в надёжное место** (например, в заметках). Она вам нужна будет дважды: для GitHub Actions и для Vercel.

> Если страница закрылась — найдёте строку в Neon: **Dashboard → ваш проект → Connection Details**

✅ **База данных создана.**

## ШАГ 3. Создать Telegram-бот

### 3.1. Создать бота

1. Откройте Telegram, найдите **@BotFather** (с галочкой "официальный аккаунт")
2. Напишите `/newbot`
3. BotFather спросит имя бота — напишите `Yí заказы` (любое читаемое)
4. Потом username — должен заканчиваться на `bot`, например `yi_orders_bot`
5. BotFather пришлёт сообщение с токеном вида:
   ```
   1234567890:AAEEEEEEEEEEEEEEEEEEEEEEEEEEEEEEEEE
   ```
6. **🚨 Скопируйте токен и сохраните в заметки.**

### 3.2. Узнать ваш chat_id

1. В Telegram найдите **@userinfobot**
2. Напишите ему `/start`
3. Он пришлёт ваше ID — число вида `123456789`
4. **🚨 Сохраните этот ID в заметки.**

### 3.3. Активировать вашего бота

1. Найдите своего нового бота в Telegram (по username, который вы придумали)
2. Напишите ему `/start`

Это нужно один раз — чтобы бот мог отправлять вам сообщения.

✅ **Telegram готов.** В заметках у вас должны быть:
- DATABASE_URL (от Neon)
- TELEGRAM_BOT_TOKEN
- TELEGRAM_MANAGER_CHAT_ID

## ШАГ 4. Загрузить данные в базу через GitHub Actions

Сейчас в Neon база пустая. Нужно создать таблицы и залить 1951 товар из Excel. Это делается одной кнопкой через GitHub Actions.

### 4.1. Добавить DATABASE_URL в секреты GitHub

1. Откройте свой репозиторий на GitHub
2. Сверху **Settings** → слева **Secrets and variables** → **Actions**
3. Зелёная кнопка **New repository secret**
4. **Name:** `DATABASE_URL`
5. **Secret:** вставьте Connection string от Neon (полностью)
6. **Add secret**

### 4.2. Запустить setup

1. Сверху на странице репозитория **Actions**
2. Если увидите сообщение про "Workflows aren't being run on this forked repository" — нажмите кнопку **I understand my workflows, go ahead and enable them**
3. Слева в списке выберите **Setup Database**
4. Справа кнопка **Run workflow** (синяя)
5. Появится выпадашка — оставьте значение **full** → **Run workflow**

Подождите 2-3 минуты, страница обновится. Должна появиться зелёная галочка ✅.

Кликните на запуск → увидите шаги:
- Push DB schema ✅
- Seed cities ✅
- Seed categories ✅
- Import products from Excel ✅ (там будет лог "Created: 1951")

Если что-то красное (❌) — кликните на ошибку, скопируйте сообщение и напишите мне.

✅ **База заполнена!**

## ШАГ 5. Развернуть сайт на Vercel

### 5.1. Регистрация

1. **Включите VPN** (на всякий случай — без него Vercel может не открываться)
2. Откройте [vercel.com](https://vercel.com/)
3. **Sign Up** → войдите через **GitHub** (так проще всего)
4. Vercel попросит разрешение на доступ к вашим репозиториям — разрешите

### 5.2. Импортировать проект

1. После входа на главной Vercel будет кнопка **Add New... → Project**
2. В списке репозиториев найдите `yi-marketplace` → **Import**
3. Откроется страница настройки. Vercel сам определит что это Next.js — не меняйте Framework Preset
4. **Раскройте секцию "Environment Variables"** — здесь нужно вставить все переменные:

Добавьте каждую переменную **по одной**:

| Key | Value |
|---|---|
| `DATABASE_URL` | (вставьте Connection string от Neon — ту же что в GitHub Secret) |
| `AUTH_SECRET` | сгенерируйте на [generate-secret.vercel.app/32](https://generate-secret.vercel.app/32) → скопируйте → вставьте |
| `TELEGRAM_BOT_TOKEN` | (ваш токен от BotFather) |
| `TELEGRAM_MANAGER_CHAT_ID` | (ваш ID от userinfobot) |

> S3-переменные пока пропускаем — без них всё работает, кроме формы подбора с фото. Подключим позже когда настроим Selectel.

5. Нажмите большую кнопку **Deploy**

Vercel начнёт сборку — это займёт 2-4 минуты. Появится экран с прогрессом, потом — конфетти 🎉 и ссылка на ваш сайт.

### 5.3. Открыть сайт

После деплоя нажмите на превью или ссылку — откроется ваш сайт в виде:
```
https://yi-marketplace-abc123.vercel.app
```

✅ **Сайт работает в интернете!**

## ШАГ 7. Подключить свой домен (опционально)

Когда купите домен (например `yi-opt.ru`):

1. На Vercel → ваш проект → **Settings → Domains**
2. Введите домен → **Add**
3. Vercel покажет DNS-записи, которые нужно прописать у вашего регистратора домена (Reg.ru, Beget):
   - тип `A`, значение типа `76.76.21.21`
   - тип `CNAME` для `www` → `cname.vercel-dns.com`
4. Зайдите в кабинет регистратора домена, добавьте эти записи
5. Через 5-30 минут Vercel автоматически активирует домен + SSL

Также добавьте в Environment Variables на Vercel:
- `AUTH_URL` = `https://yi-opt.ru` (ваш домен)

Vercel автоматически передеплоит сайт с новой переменной.

## 🆘 Что может пойти не так

### "Я не могу зайти на Vercel"
Включите VPN. Самый простой — расширение Browsec в Chrome (бесплатно).

### "GitHub Actions упал на шаге Import products"
Проверьте, что Excel-файл `мебель_1688.xlsx` есть в репозитории по пути `data/мебель_1688.xlsx`. Откройте репозиторий на GitHub, кликните `data` — там должен быть файл. Если нет — добавьте через GitHub Desktop.

### "Сайт открылся, но фото товаров не грузятся"
Это нормальная задержка при первом открытии — Vercel прокачивает изображения через `/api/img-proxy`. Подождите минуту, перезагрузите страницу.

### "В Telegram не приходит уведомление о заказе"
Три причины обычно:
1. Не отправили `/start` своему боту — сходите в Telegram, найдите бота, нажмите Start
2. Неправильный токен или chat_id — пересохраните в Vercel Environment Variables
3. Vercel ещё не пересобрал с новыми переменными — на Vercel → Deployments → последний → **Redeploy**

### "Ошибка 500 при оформлении заказа"
В Vercel → ваш проект → **Logs** в верхнем меню — там будет красная строка с ошибкой. Скопируйте её и напишите мне.

### "Я сделал ошибку в Environment Variables, как изменить?"
Vercel → ваш проект → **Settings → Environment Variables** → найдите переменную → нажмите три точки → **Edit**. После изменения нужно сделать Redeploy: **Deployments → последний → ⋮ → Redeploy**.
